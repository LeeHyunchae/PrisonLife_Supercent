확정된 게임 룰:

- **자원 흡수 단위**: 1개씩 차감 + 1개씩 연출. 묶음 처리 안 함 (도중 이탈 시 데이터 일관성).
- **무기 강화**: 누적 3단계
  - 단계 0: Pickaxe (곡괭이)
  - 단계 1: HandDrill (핸드드릴)
  - 단계 2: Drill (드릴)
  - 단계 3: RockCrusher (암석파쇄차)
  - 강화 시 **채굴속도 + 등 광석 한도 동시 증가**. 데이터는 `PlayerStatsConfigSO` 에 단계별로 정의.
- **광부 일꾼 구매**: 1회 구매 → **3명 동시 생성** → 구매칸 사라짐. (영상 검증)
- **감옥 확장**: 1회 구매 → 시네마 → **엔드카드** (게임 종료 트리거)
- **죄수 처리 일꾼**: **MVP 제외**. (영상에 없음. Gate 코드는 후속 확장 위해 작성됨)
- **광석 재생성**: HP 0 → 3~5초 후 같은 자리 재스폰. (스폰 풀 사용)
- **죄수 스폰**: 빈 슬롯 즉시 채움, 풀 재활용.
- **돈 단위**: 죄수 1명 처리 = 돈 아이템 1개 (=N원). 후속 조정 가능.
- **적재 한도 초기값** (인스펙터 SerializeField 로 노출, SO 도입 후 이동 예정):
  - 등 광석 10 / 등 돈 30 / 손 수갑 6
  - 컨테이너 저장 광석 20 / 출력 수갑 8
  - 수갑 생산 주기 1.0 초

**Why:** 사용자가 Prison Life 영상을 보고 직접 검증한 내용. 일부는 기획서와 다름(예: 광부 1회 구매로 3명 동시 = 영상 기준).

**How to apply:** 시설/액터 동작 구현 시 위 룰 준수. 룰이 모호한 부분(돈 단위 등)은 사용자 확인 후 진행. 영상과 기획서가 충돌하면 **영상 우선**.
