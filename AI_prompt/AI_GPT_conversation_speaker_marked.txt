5-1. 길 그리기
public class PathDrawer : MonoBehaviour
{
    [SerializeField] private GameObject cubePrefab;
    [SerializeField] private LayerMask groundMask;
    private List<Vector3Int> _path = new();

    public void TryAddBlock(Vector3 worldPos)
    {
        var cell = Vector3Int.RoundToInt(worldPos);
        if (_path.Contains(cell)) return;
        Instantiate(cubePrefab, cell + Vector3.up * 0.5f, Quaternion.identity, pathRoot);
        _path.Add(cell);
        UIManager.UpdateRemaining(maxBlocks - _path.Count);
    }
}

제가 다시 본 결론부터 말하면, 이 과제 영상은 Prison Life 본게임 전체라기보다는, 하이퍼캐주얼 광고/플레이어블 광고 스타일로 압축된 “교도소 확장 + 자원 수집 + 시설 해금” 데모에 가깝습니다.

1. 전체 게임 장르
3D 아이소메트릭 방치형 타이쿤 / 캐주얼 매니지먼트 게임입니다.

플레이어는 파란 제복의 교도관 캐릭터를 조이스틱으로 움직이고, 맵 곳곳에서 다음 행동을 반복합니다.

바위/장애물 제거

돈 수집

특정 설치 구역에 돈 지불

기계/시설/감방/작업 구역 해금

죄수들이 시설을 사용하거나 노동함

더 많은 돈을 벌고 다음 구역을 확장함

공식 설명에서도 “죄수 관리, 시설 업그레이드, 직원 관리, 교도소 확장”이 핵심으로 소개되어 있고, 영상은 그중에서도 광산/채굴/감방 확장/체크인 데스크/경찰/죄수 흐름을 보여주는 튜토리얼성 장면입니다. 

2. 영상에서 보이는 핵심 UI
고정 UI
화면은 세로형 720×1280 기준입니다.

위치	요소	설명
좌상단	PRISON LIFE 로고 + PLAY 버튼	광고/플레이어블 광고용 오버레이처럼 보입니다. 실제 과제에서는 로고/플레이 버튼까지 재현하면 좋습니다.
우상단	돈 UI	초록 지폐 아이콘 + 현재 보유금. 0, 30, 80, 135, 235처럼 계속 변합니다.
좌하단	사운드 버튼	파란 원형 스피커 버튼.

월드 위	파란 손가락 포인터	다음에 가야 할 위치를 안내하는 튜토리얼 포인터.
3. 플레이어 조작
조작은 단순합니다.

가상 조이스틱으로 교도관 캐릭터를 이동합니다.
플레이어가 특정 오브젝트 근처 또는 지정 영역 안에 들어가면 자동 상호작용이 발생합니다.

영상상 직접 버튼을 눌러 채굴하거나 설치하는 방식은 아니고, 대부분은 근처에 서 있으면 자동으로 진행되는 하이퍼캐주얼 방식입니다.

예를 들면:

상황	플레이어 행동	자동 발생
바위 더미 근처로 이동	조이스틱 이동	곡괭이질/채굴 애니메이션, 바위 파괴
돈 더미 근처로 이동	접근	돈 자동 수집
흰색 점선 설치 구역으로 이동	진입	보유 돈이 차감되며 시설 건설
책상/데스크 근처 이동	접근	수갑/체크인 요구량 처리
감방 근처 이동	접근	죄수 수용/카운트 증가
4. 게임 루프 상세 분석
영상에서 반복적으로 보이는 루프는 이겁니다.

조이스틱으로 이동
→ 바위 제거 또는 죄수 노동으로 자원 획득
→ 돈/재료를 들고 이동
→ 설치 구역에 지불
→ 시설이 생성됨
→ 더 많은 죄수/노동/수익이 발생
→ 다음 구역 확장
과제 구현에서는 이 루프를 잘 살리는 게 핵심입니다.

5. 오브젝트별 상세 분석
5-1. 플레이어: 파란 제복 교도관
영상의 주 조작 캐릭터입니다.

특징:

파란 경찰 모자

파란 상의

작은 3D 캐릭터

조이스틱으로 이동

바위 근처에서 곡괭이질

돈/자원을 등에 쌓아 들고 다님

자원 스택이 높아지면 캐릭터 뒤/옆으로 초록색 지폐 기둥처럼 표현됨

너무 많이 들면 MAX 텍스트가 뜸

구현 포인트는 캐릭터 이동 + 소지 자원 스택 표시 + 근접 자동 상호작용입니다.

5-2. 자원 스택
영상에서 가장 많이 보이는 핵심 표현입니다.

플레이어가 돈을 먹으면 캐릭터 뒤쪽에 초록색 지폐 블록이 세로로 쌓입니다.
나중에는 회색/검정색 기둥 같은 자원도 들고 다닙니다.

특징:

돈을 먹으면 상단 보유금 UI가 증가하거나, 캐릭터가 들고 있는 스택이 증가

특정 설치 구역에 들어가면 스택이 줄어들고 시설에 투입됨

스택 최대치에 도달하면 캐릭터 위에 MAX 표시

스택은 약간 휘어져 보이기도 함. 과제에서는 단순 세로 스택으로 구현해도 충분합니다.

구현은 어렵게 하지 말고:

현재 소지금 / 최대 소지금
→ 소지금 개수에 따라 StackVisual 자식 오브젝트 활성화
→ 최대치면 MAX 텍스트 활성화
이 정도면 영상 느낌이 납니다.

5-3. 바위 / 암석 지대
초반부의 큰 핵심입니다.

영상 시작 시 맵 위쪽에 거대한 바위 더미가 막고 있습니다.
플레이어가 바위 더미로 들어가면 곡괭이질을 하면서 바위가 조금씩 부서지고 길이 열립니다.

특징:

바위는 여러 개의 회색/남색 저폴리 암석 덩어리

플레이어가 가까이 있으면 자동으로 채굴

채굴 시 작은 파편 이펙트

길이 점점 뚫림

중간에 노란 안전모 NPC 죄수들이 함께 채굴하는 장면도 나옴

지게차/드릴 차량이 등장해 바위를 빠르게 제거하는 장면도 있음

구현 우선순위는 높습니다.
이 영상의 “확장감”이 바위 제거에서 나오기 때문입니다.

MVP 구현 방식:

바위를 개별 Rock 오브젝트 여러 개로 배치

각 Rock에 HP 부여

플레이어가 일정 거리 안에 있으면 0.2초마다 HP 감소

HP 0이 되면 파편 이펙트 후 비활성화

Rock 제거 시 돈 드랍 또는 즉시 돈 지급

5-4. 돈 묶음 / 초록 지폐
영상 곳곳에 초록색 돈 묶음이 있습니다.

종류가 두 가지로 보입니다.

바닥에 놓인 돈 묶음

캐릭터가 들고 다니는 세로 스택

바닥 돈은 플레이어가 접근하면 자동 수집됩니다.
돈 더미는 흰색 테두리 패드 위에 놓여 있기도 하고, 바위 제거 후 드랍되기도 합니다.

구현 포인트:

MoneyPickup

플레이어 접촉 시 보유금 증가

UI 숫자 갱신

돈이 캐릭터 쪽으로 날아오는 간단한 Tween 이펙트가 있으면 완성도 상승

5-5. 흰색 점선 설치 구역
영상에서 매우 중요합니다.

바닥에 흰색 점선 사각형 또는 흰색 테두리 패드가 있고, 플레이어가 그곳에 들어가면 돈을 소비해 시설이 설치됩니다.

예시:

책상 설치 구역

기계 설치 구역

광산/작업대 설치 구역

감방 확장 구역

특정 시설 구매 패드

일부 패드에는 비용이 표시됩니다.

예를 들면 영상에 50으로 보이는 구역들이 있습니다.
플레이어가 충분한 돈을 가지고 들어가면 돈 스택이 줄어들고, 시설이 생성되거나 업그레이드됩니다.

구현 방식:

BuildZone
- cost: 50
- targetPrefab: Desk / Machine / Cell / WorkStation
- isBuilt: false

Player가 영역 안에 있음
→ 일정 속도로 돈 차감
→ cost가 0이 되면 prefab 활성화
이게 이 장르에서 매우 흔한 방식이고, 영상과도 잘 맞습니다.

5-6. 체크인 데스크 / 수갑 요구 UI
30초 부근에 책상과 모니터가 등장하고, 옆에 일반인/수감자처럼 보이는 NPC들이 줄을 서 있습니다.
말풍선에는 수갑 아이콘 + 숫자 4가 표시됩니다.

이건 아마:

죄수 입소 처리

수갑 자원 필요

책상/접수 시설에서 죄수 등록

죄수 4명 처리 필요

같은 기능으로 보입니다.

과제에서 전부 구현하기보다는, 간단히 이렇게 구현하면 됩니다.

NPC들이 줄 서 있음
→ 플레이어가 데스크 근처에 가면 처리 진행
→ 일정 시간 후 죄수가 감방/작업장으로 이동
→ 보상 돈 지급
수갑 자원까지 구현하면 좋지만, MVP에서는 “데스크 처리 → 죄수 생성” 정도로 충분합니다.

5-7. 죄수 NPC
영상에는 주황색 죄수복을 입은 NPC들이 계속 등장합니다.

종류가 있어 보입니다.

죄수 형태	역할
일반 주황 죄수	맵을 이동하거나 감방으로 들어감
노란 안전모 죄수	광산/바위 제거 작업을 함
감방 안 죄수	수용 인원 카운트로 표시됨
줄 서 있는 NPC	접수/수감 대기자로 보임
중요한 건 “죄수들이 자동으로 움직여서 교도소가 살아 있는 것처럼 보인다”는 점입니다.

구현 우선순위:

죄수 프리팹 생성

지정 위치로 이동

감방에 들어가면 카운트 증가

작업 죄수는 바위 근처에서 반복 애니메이션

5-8. 감방
초반 화면 아래쪽에 감방이 있습니다.

안쪽 벽에 0/20, 이후 3/20, 12/20 같은 수용 인원 표시가 보입니다.

역할:

죄수 수용 공간

죄수가 들어가면 카운트 증가

특정 수 이상 수용하면 돈 생산 또는 다음 시설 해금 가능

감방 안에 죄수들이 주황색 옷으로 여러 명 서 있음

구현 방식:

CellRoom
- capacity: 20
- currentCount
- inmateSlots[]
- currentCount / capacity UI 표시
죄수가 들어오면 슬롯 위치에 죄수 모델을 배치하고 카운트를 올리면 됩니다.

5-9. 기계 / 컨베이어 / 광산 시설
영상 중반부터 파란색 기계, 컨베이어, 수갑 같은 아이콘, 회색 자원 더미가 등장합니다.

기계는 다음 기능을 하는 것으로 보입니다.

광산/채굴 관련 자원 생성

수갑/장비 생산

죄수 처리 또는 입소 처리에 필요한 자원 생성

업그레이드/구매 대상

영상상 완전히 복잡한 생산 체인을 구현할 필요는 없어 보입니다.
과제에서는 간단히:

기계가 일정 시간마다 돈 또는 수갑 자원 생성
→ 플레이어가 근처에 가면 수집
→ 데스크/감방/채굴 작업에 사용
정도로 구현하면 영상과 비슷합니다.

5-10. 직원 / 교도관 NPC
플레이어 외에도 검은 옷/남색 제복의 경찰 NPC가 돌아다닙니다.

역할은 정확히 플레이어처럼 자원을 들고 다니는 자동 작업자일 수도 있고, 단순 배경 NPC일 수도 있습니다.

공식 설명에는 “staff management”, 즉 직원을 고용하고 관리하는 요소가 언급됩니다. 

따라서 영상의 경찰 NPC는 “직원/자동 작업자”로 보는 게 자연스럽습니다.

과제에서 구현하면 좋은 요소:

일정 비용 지불 시 직원 1명 생성

직원은 자동으로 돈 수집 또는 죄수 운반

플레이어가 직접 하지 않아도 일부 작업 진행

하지만 이건 후순위입니다. 먼저 플레이어 직접 루프가 돌아가야 합니다.

5-11. 지게차 / 드릴 차량
90초 이후부터 노란 지게차/드릴 차량이 등장합니다.

기능:

큰 바위 지대를 빠르게 파괴

플레이어가 근처에서 자원을 공급하거나, 차량이 활성화되면 자동으로 전진

채굴 이펙트가 훨씬 강함

후반 확장 구역을 여는 고급 시설처럼 보임

과제에서는 시간이 부족하면 생략해도 됩니다.
넣는다면 “후반부 연출용”으로 좋습니다.

간단 구현:

DrillVehicle
- 활성화 전: BuildZone에 비용 투입
- 활성화 후: 앞으로 이동
- 경로상의 Rock을 빠르게 파괴
6. 영상의 시간대별 흐름
제가 프레임을 다시 뽑아 보니 대략 이런 순서입니다.

시간대	내용
0~4초	튜토리얼. 조이스틱 드래그 안내. 플레이어는 바위 지대 앞에 있음. 감방에는 0/20 표시.
5~18초	플레이어가 바위 지대로 이동해 채굴. 바위가 부서지고 길이 열림. MAX 표시가 뜨는 것으로 보아 자원 소지 제한 존재.
20~26초	바위 너머의 기계/생산 구역으로 이동. 회색 자원 더미, 파란 기계, 설치 예정 구역이 보임.
28~36초	데스크/돈 더미/수감자 줄 등장. 플레이어가 돈을 수집하고 초록 돈 스택을 들기 시작.
38~46초	바위 지대에서 채굴 반복. 돈을 사용해 구역/시설 해금.
50~64초	죄수 작업자들이 바위 근처에서 함께 채굴. 플레이어가 돈 스택을 들고 돌아다님.
66~80초	기계/시설에 자원 투입. 회색 기둥/검은 기둥 형태의 자원 스택이 등장.
82~90초	돈 수집 → 상단 재화 증가. 플레이어가 큰 스택을 들고 다시 바위 지대 이동.
92~100초	지게차/드릴 차량이 바위를 뚫음. 후반 확장 연출.
102~120초	돈 스택 대량 수집, 시설/광산/감방 주변 이동, 큰 자원 스택 MAX 표시.
122초 이후	감방에 죄수 다수 수용. 하단에 No Cell! 말풍선도 보여서 감방 수용 한계/추가 감방 필요 상태가 있는 것으로 보임.
125초 이후	광고식 엔드카드. Prison Life 로고 + 앱 아이콘 + CONTINUE 버튼.

8. 과제용 MVP 설계
제가 보기엔 다음 정도를 목표로 잡는 게 좋습니다.

목표 플레이 흐름
1. 시작
   - 플레이어가 감방 앞에서 시작
   - 감방은 0/20
   - 바위가 길을 막고 있음
   - 조이스틱 튜토리얼 표시

2. 바위 채굴
   - 플레이어가 바위에 접근하면 자동 채굴
   - 바위가 깨지면서 돈 드랍
   - 돈을 먹으면 등에 초록 스택이 쌓임

3. 데스크 건설
   - 돈 50을 BuildZone에 넣으면 접수 데스크 생성
   - 이후 죄수 NPC가 줄 서거나 생성됨

4. 감방 수용
   - 플레이어가 죄수/수갑/데스크 처리 후 감방으로 보내면 1/20 증가
   - 감방에 죄수 모델이 하나씩 배치됨

5. 채굴 작업자 해금
   - 돈 100을 지불하면 노란 안전모 죄수 작업자 생성
   - 작업자가 자동으로 바위 HP를 깎음

6. 다음 구역 해금
   - 큰 바위 벽을 제거하면 새 영역이 열림
   - 새 기계/돈 더미/구매 패드 등장

7. 엔드
   - 일정 시설 3개 이상 해금 또는 감방 10명 수용 시 Clear/Continue 팝업
이 정도면 과제 영상의 핵심을 꽤 잘 따라갑니다.

11. 구현에서 가장 중요한 디테일
돈 스택은 꼭 넣는 게 좋습니다
영상에서 “내가 돈을 많이 들고 있다”는 게 세로 스택으로 강하게 보입니다.
과제 평가자도 영상과 비교할 때 바로 알아볼 요소입니다.

단순 구현:

moneyStackRoot 아래에 moneyBlock 20개 미리 배치
보유량 비율에 따라 0~20개 활성화
보유량이 max면 MAX 텍스트 표시
바위는 “한 덩어리 벽”보다 “여러 개의 작은 바위”가 낫습니다
한 번에 사라지면 느낌이 약합니다.
영상처럼 길이 조금씩 뚫리는 느낌을 내려면 바위 프리팹을 여러 개 깔고 하나씩 제거하세요.

BuildZone은 흰색 테두리 + 비용 표시가 핵심입니다
하이퍼캐주얼 타이쿤 문법입니다.
바닥에 흰색 네모 패드, 위에 50, 100 같은 비용 표시를 띄우면 영상 느낌이 바로 납니다.

죄수는 복잡한 AI보다 “줄 서기 + 감방 이동”만 해도 충분합니다
죄수 AI를 복잡하게 만들 필요 없습니다.

SpawnPoint → DeskQueuePoint → CellDoor → CellSlot
이 경로만 따라가도 화면이 살아 보입니다.

## Turn 6 - 사용자

우선 캐릭터는 터치형 조이스틱으로 이동해 조이스틱은 화면의 어디를 눌러도 누른 자리에서 등장하면서 해당 지점을 기점으로 손가락을 이동한 방향으로 캐릭터를 이동시켜
캐릭터는 곡괭이를들고(돈을 주고 다른 무기?로 강화가 가능한) 광석을 캐서 광석을 등에 짊어져 광석 하나당 한개씩 짊어지는 것 같아
광석을 짊어진 캐릭터가 수갑 컨테이너의 상호작용 칸(흰 점선 네모박스)에 들어가면 광석이 컨테이너 오브젝트로 빨려들어가고 컨테이너의 아이템 출력 공간에서 일정 기간에 한번씩 광석을 소모하면서 아이템을 생산해 생상한 아이템(수갑)에 캐릭터가 가까이가면 수감이 빨려들어와서 캐릭터가 손에 들어(이것도 광석처럼 탑형태로 쌓이지만 등이 아니라 손에 들어)
수갑을 든 채로 죄수? 캐릭터들이 나란히 줄 서 있는곳에 있는 죄수 상호작용 칸에 들어가면 죄수마다 필요한 수갑(아마 1~4정도의 랜덤인듯) 만큼 손에 있는 수갑이 죄수에게 빨려들어가고 상호작용 칸 옆에있는 돈 생성장소에 돈이 생성되며 죄수는 죄수복으로 갈아입고 감옥으로 호송돼
돈 생성장소에 캐릭터가 접근하면 돈을 탑형태로 등에 짊어지고, 이를 통해서 무기를 구매하거나 대신 광석을 캐는 일꾼을 고용할 수 있어 
일꾼은 캐릭터가 가장 처음에 가졌던 곡괭이를 통해서 광석을 캐고 캐어진 광석은 자동으로 수갑 생성 컨테이너 상호작용칸으로 빨려들어가서 탑처럼 쌓여
죄수 상호작용 칸에 추가로 구매 칸이 열리는데 구매칸은 죄수 상호작용 일꾼이야
일꾼은 수갑 컨테이너에서 수갑을 들어서 죄수 상호작용칸에 가져다 놓고, 해당 일꾼이 죄수 상호작용 칸으로 돌아가면 죄수에게 수갑을 주고 돈 생성칸에 돈을 벌어놔
죄수가 호송되는 감옥은 최대 수용 인원이 정해져있고, 최대 수용인원에 도달하면 감옥 크기를 늘릴 수 있는 구매 칸이 생성돼, 구매칸에 캐릭터가 들어가면 돈이 들어가고 감옥이 넓어져

대략 이게 이 영상의 흐름임

## ChatGPT 응답

네, 이 설명 기준으로 보면 흐름이 훨씬 명확해졌습니다.
이 게임은 단순 “돈 모아서 시설 해금”보다, 실제로는 광석 → 수갑 → 죄수 처리 → 돈 → 업그레이드/고용/확장으로 이어지는 미니 생산 체인 게임이네요.

핵심 루프는 이렇게 잡으면 됩니다.

광석 채굴
→ 광석을 등에 적재
→ 수갑 컨테이너에 광석 투입
→ 컨테이너가 광석을 소비해 수갑 생산
→ 수갑을 손에 적재
→ 죄수 처리 구역에 수갑 납품
→ 죄수가 죄수복으로 변경되고 감옥으로 이동
→ 돈 생성
→ 돈 수집
→ 무기 강화 / 광부 고용 / 죄수 처리 일꾼 고용 / 감옥 확장
그리고 말씀하신 것처럼 이 구조는 오브젝트 풀 + 자원 운반 인터랙션만 제대로 잡으면 거의 대부분 구현 가능합니다.

1. 자원 종류 정리
영상 기준 자원은 최소 3종류입니다.

자원	시각적 위치	획득처	사용처
광석	캐릭터 등	광석 채굴	수갑 컨테이너 투입
수갑	캐릭터 손	수갑 컨테이너 생산물	죄수 처리

중요한 점은 광석과 돈은 등에 쌓이고, 수갑은 손에 쌓인다는 점입니다.
그래서 인벤토리를 단순히 int money 하나로 두기보다는, 최소한 “운반 슬롯”을 나눠야 합니다.

BackCarrySlot  : Ore 또는 Money
HandCarrySlot  : Handcuff
단, 등에 광석과 돈을 동시에 들 수 있는지는 영상에서 애매하지만, 구현은 등 슬롯에는 한 종류만 들 수 있음으로 제한하는 게 깔끔합니다.

7. 죄수 처리 로직
죄수는 각각 필요한 수갑 수가 다릅니다.

Prisoner
- requiredHandcuffs: 1~4
- currentHandcuffs: 0
- state: Waiting / Processing / Escorted / InCell
처리 흐름:

죄수 처리칸에 캐릭터가 들어감
→ 캐릭터 손의 수갑을 1개씩 죄수에게 흡수
→ currentHandcuffs 증가
→ requiredHandcuffs 충족
→ 죄수복으로 변경
→ 감옥으로 이동
→ 돈 생성장소에 돈 생성
중요한 건 수갑이 한 번에 사라지지 않고, 하나씩 빨려 들어가는 연출입니다.
이것도 풀링된 HandcuffItem을 잠깐 생성해서 죄수 쪽으로 이동시키면 영상 느낌이 납니다.

8. 감옥 수용량 / 확장
감옥은 단순히 카운트로 관리하면 됩니다.

PrisonCell
- currentCount
- capacity
- prisonerSlots
- expandCost
흐름:

죄수 처리 완료
→ 감옥에 빈자리 있는지 확인
→ 있으면 지정 슬롯으로 이동
→ currentCount 증가
→ currentCount == capacity가 되면 No Cell 또는 ExpandZone 활성화
확장 구매:

ExpandZone 활성화
→ 플레이어가 돈 들고 진입
→ 돈이 하나씩 투입
→ 비용 충족
→ 감옥 Mesh/울타리 확장
→ capacity 증가
→ 새 prisonerSlots 활성화
→ ExpandZone 비활성화
MVP에서는 실제로 감옥 크기를 늘리지 않고, 울타리 일부 활성화 + 슬롯 증가 + 표시 숫자 변경만 해도 충분합니다.

9. 수갑 컨테이너 로직
컨테이너가 꽤 중요합니다.

HandcuffContainer
- storedOre
- maxStoredOre
- producedHandcuffs
- maxProducedHandcuffs
- productionInterval
동작:

광석 투입칸에 carrier가 들어옴
→ carrier의 Ore를 1개씩 컨테이너로 이동
→ storedOre 증가

생산 루프
→ storedOre > 0 이고 output 공간이 가득 차지 않았으면
→ 일정 시간마다 storedOre -1
→ producedHandcuffs +1
→ 출력 공간에 HandcuffItem 생성
출력 공간:

캐릭터가 가까이 감
→ producedHandcuffs를 1개씩 캐릭터 손으로 이동
→ handAmount 증가
여기서도 컨테이너 내부 저장량을 시각화할지 선택해야 합니다. 영상에는 광석이 컨테이너에 탑처럼 쌓이는 느낌이 있으니, storedOre만큼 컨테이너 위/옆에 StackVisual을 표시하면 좋습니다.

14. 가장 중요한 설계 포인트

15. 과제 제출에서 어필할 수 있는 부분
README에 이런 식으로 쓰면 개발 역량이 잘 보일 겁니다.

- 영상의 핵심 루프를 광석 → 수갑 → 죄수 처리 → 돈 → 확장 생산 체인으로 분석했습니다.
- 반복 생성되는 광석/수갑/돈/죄수/이펙트는 Object Pool 기반으로 관리했습니다.
- 플레이어와 일꾼은 공통 ResourceCarrier를 사용하여 자원 운반 로직을 재사용했습니다.
- 모든 상호작용은 InteractionZone 기반의 자동 흡수/투입 구조로 구현했습니다.
- AI는 영상 분석, 시스템 구조화, 코드 초안 작성, 간단한 리소스 제작에 활용했습니다.
이 방향이면 그냥 겉모양만 따라 한 게 아니라, 게임의 내부 루프를 제대로 이해하고 구조화했다는 느낌이 납니다.

죄수 처리 일꾼 구매칸	돈을 지불해 죄수 처리 일꾼을 고용하는 구역
동작 흐름
죄수가 대기열에 서 있음
→ 각 죄수는 필요한 수갑 수량을 가짐
→ 플레이어 또는 일꾼이 수갑을 들고 처리칸에 진입
→ 수갑이 죄수에게 하나씩 흡수됨
→ 필요 수갑 수량 충족
→ 죄수가 죄수복으로 변경됨
→ 감옥으로 이동
→ 돈 생성장소에 돈 생성
6.4 돈 생성장소
역할
죄수 처리 완료 후 돈이 생성되는 장소이다.

동작
죄수 처리 완료
→ 돈 생성장소에 돈 아이템 생성
→ 플레이어가 접근
→ 돈이 플레이어 등으로 흡수
→ 플레이어의 돈 적재량 증가
돈은 플레이어가 등에 직접 짊어지고, 구매칸에서 사용한다.

6.5 구매칸
역할
돈을 투입해 특정 기능을 해금하거나 업그레이드하는 구역이다.

공통 동작
플레이어가 돈을 들고 구매칸에 진입
→ 돈이 하나씩 구매칸으로 흡수
→ 남은 비용 감소
→ 비용을 모두 지불하면 구매 완료
→ 지정된 기능 실행
구매칸 종류
구매칸	결과
무기 강화 구매칸	플레이어 채굴 속도 증가, 무기 외형 변경
광부 일꾼 구매칸	광부 일꾼 생성
죄수 처리 일꾼 구매칸	죄수 처리 일꾼 생성
감옥 확장 구매칸	감옥 수용 인원 증가, 감옥 크기 확장
6.6 감옥
역할
처리 완료된 죄수를 수용하는 공간이다.

주요 속성
속성	설명
현재 수용 인원	현재 감옥에 들어간 죄수 수
최대 수용 인원	감옥이 수용 가능한 최대 죄수 수
죄수 배치 슬롯	죄수가 감옥 안에 배치될 위치
확장 비용	감옥 확장에 필요한 돈
흐름
죄수 처리 완료
→ 감옥에 빈 슬롯 확인
→ 빈 슬롯이 있으면 죄수 이동
→ 현재 수용 인원 증가
→ 최대 수용 인원 도달
→ 감옥 확장 구매칸 활성화
최대 수용 인원 도달 시

8. 오브젝트 풀 설계
본 게임은 반복적으로 생성/제거되는 오브젝트가 많기 때문에, Instantiate와 Destroy를 직접 반복하지 않고 오브젝트 풀을 사용한다.

8.2 풀 매니저 역할

18. 최종 구현 목표 요약
이 과제에서 최종적으로 보여줘야 할 것은 다음이다.

플레이어가 이동한다.
광석을 캔다.
광석을 등에 쌓는다.
광석을 컨테이너에 넣는다.
컨테이너가 수갑을 만든다.
수갑을 손에 든다.
죄수에게 수갑을 준다.
죄수가 감옥으로 이동한다.
돈이 생긴다.
돈을 등에 쌓는다.
돈으로 장비를 강화하거나 일꾼을 고용하거나 감옥을 확장한다.
이 흐름이 자연스럽게 보이면, 영상 속 게임의 핵심 구조를 충분히 재현한 것으로 볼 수 있습니다.
