Prison Life 모작 과제 개발기획서
1. 프로젝트 개요
1.1 프로젝트 목적

본 프로젝트는 과제 영상에 등장하는 Prison Life 스타일의 하이퍼캐주얼 방치형 타이쿤 게임을 Unity로 모작하는 것을 목표로 한다.

플레이어는 교도관 캐릭터를 조작하여 광석을 채굴하고, 광석을 수갑 컨테이너에 투입하여 수갑을 생산한다. 생산된 수갑으로 죄수를 처리하면 돈을 얻을 수 있으며, 획득한 돈으로 무기 강화, 일꾼 고용, 감옥 확장 등을 진행한다.

3. 조작 방식
3.1 터치형 조이스틱

플레이어는 화면의 아무 위치나 터치할 수 있다.
터치가 시작된 위치에 가상 조이스틱이 생성되며, 손가락을 이동한 방향으로 캐릭터가 움직인다.

3.2 조이스틱 동작 방식
입력	동작
화면 터치 시작	터치 위치에 조이스틱 표시
드래그	드래그 방향으로 캐릭터 이동
터치 유지	캐릭터 계속 이동
터치 종료	조이스틱 숨김, 캐릭터 정지
3.3 상호작용 방식

캐릭터가 특정 상호작용 영역에 들어가면 자동으로 동작한다.

5.1.1 무기 단계 (영상 분석 추가)
플레이어는 무기 강화 구매칸에서 단계적으로 무기를 교체한다. 무기에 따라 채굴 성능과 등 광석 한도가 변한다.
광석은 HP 가 없고 한 번의 스윙 임팩트에 닿으면 무조건 부서진다. 무기별 차이는 스윙 길이(채굴 속도)·1회 타격 수·박스 범위·등 광석 한도이다.
총 3단계 (강화 2회).
단계	무기	스윙 길이(s)	1회 타격 수	박스 W	박스 D	등 광석 한도
0	곡괭이	0.7	1	1.0	1.0	10
1	핸드드릴	0.4	2	1.5	1.2	25
2	암석파쇄차	0.2	4	3.0	2.0	40
(수치는 영상 분석 기반 1차 추정. PlayerStatsConfigSO 또는 SystemManager 인스펙터에서 조정.)
5.2 죄수
역할

죄수는 줄을 서서 대기하다가, 필요한 수갑 수량을 받으면 죄수복으로 변경되고 감옥으로 이동한다.

속성
속성	설명
필요 수갑 수	1~4개 사이의 랜덤값
현재 지급 수갑 수	지급받은 수갑 개수
상태	대기 / 처리 중 / 수감 이동 / 감옥 입장
흐름
대기열에 서 있음
→ 죄수 처리칸에서 수갑을 받음
→ 필요 수갑 수량 충족
→ 죄수복으로 변경
→ 감옥으로 이동
→ 감옥 수용 인원 증가
→ 돈 생성장소에 돈 생성
5.3 광부 일꾼
역할

광부 일꾼은 플레이어 대신 광석을 채굴한다.

특징

광부 일꾼은 광석을 직접 들고 이동하지 않는다.
광석을 채굴하는 순간, 생성된 광석이 수갑 컨테이너의 광석 적재 위치로 자동 흡수된다.

흐름
광석 위치로 이동
→ 채굴 애니메이션 반복
→ 광석 생성
→ 광석이 컨테이너 투입칸으로 빨려 들어감
→ 수갑 컨테이너의 저장 광석 증가
비주얼
노란 안전모
곡괭이 사용
채굴 시 파편 이펙트
광석이 컨테이너로 날아가는 흡수 연출
5.4 죄수 처리 일꾼
역할

죄수 처리 일꾼은 수갑 컨테이너에서 수갑을 가져와 죄수에게 지급한다.

흐름
수갑 컨테이너 출력칸으로 이동
→ 수갑을 손에 적재
→ 죄수 처리칸으로 이동
→ 죄수에게 수갑 지급
→ 돈 생성
→ 다시 수갑 출력칸으로 이동
비주얼
교도관 또는 직원 형태
손에 수갑 스택 표시
죄수 처리칸에서 수갑 납품 연출
6. 주요 시설
6.1 광석 채굴 구역
역할

플레이어와 광부 일꾼이 광석을 획득하는 장소이다.

동작
주체	동작
플레이어	광석을 채굴해 등에 적재
광부 일꾼	채굴한 광석을 즉시 컨테이너로 전송
구현 포인트
광석은 콜라이더가 트리거 (플레이어가 물리적으로 통과 가능). HP 개념 없음.
플레이어 앞에 부착된 트리거 박스(W×D)가 박스 안 광석을 채굴 후보로 등록 / 해제
박스에 후보가 1개 이상 있고 인벤토리에 여유가 있으면 채굴 스윙 애니메이션이 자동 반복 재생
스윙 1회 종료 시점에 무기 위치 기준 가장 가까운 N개 광석을 즉시 파괴 (N = 무기 단계별 1회 타격 수)
파괴된 광석은 인벤토리 +1, 광석 비주얼 비활성, 일정 시간 후 같은 자리에 재생성
박스 W×D, 스윙 길이, 1회 타격 수, 등 광석 한도는 모두 무기 단계에 따라 변동
무기 위치는 weaponAnchor (캐릭터의 손/도구 끝). 미설정 시 캐릭터 transform 폴백
6.2 수갑 컨테이너
역할

광석을 입력받아 수갑을 생산하는 핵심 생산 시설이다.

구성
구성 요소	설명
광석 투입칸	플레이어가 광석을 넣는 흰 점선 네모박스
광석 저장 스택	컨테이너에 쌓인 광석 표시
수갑 생산기	일정 시간마다 광석을 소비해 수갑 생성
수갑 출력칸	생산된 수갑이 쌓이는 공간
동작 흐름
플레이어가 광석을 들고 투입칸에 진입
→ 광석이 컨테이너로 흡수
→ 컨테이너 저장 광석 증가
→ 일정 시간마다 광석 1개 소비
→ 수갑 1개 생산
→ 수갑 출력칸에 수갑 표시
→ 플레이어 또는 죄수 처리 일꾼이 접근하면 수갑 수집
주요 속성
속성	설명
저장 광석 수	컨테이너 내부에 보관된 광석
최대 저장 광석 수	저장 가능한 최대 광석
생산된 수갑 수	출력칸에 쌓인 수갑
최대 수갑 출력 수	출력칸 최대 수량
생산 주기	수갑 1개 생산에 걸리는 시간
6.3 죄수 처리 구역
역할

대기 중인 죄수에게 수갑을 지급하고, 죄수를 감옥으로 이동시키는 장소이다.

구성
구성 요소	설명
죄수 대기열	죄수들이 줄 서 있는 위치
죄수 상호작용칸	수갑을 납품하는 흰 점선 네모박스
돈 생성장소	죄수 처리 완료 후 돈이 생성되는 위치
죄수 처리 일꾼 구매칸	돈을 지불해 죄수 처리 일꾼을 고용하는 구역
동작 흐름
죄수가 대기열에 서 있음
→ 각 죄수는 필요한 수갑 수량을 가짐
→ 플레이어 또는 일꾼이 수갑을 들고 처리칸에 진입
→ 수갑이 죄수에게 하나씩 흡수됨
→ 필요 수갑 수량 충족
→ 죄수가 죄수복으로 변경됨
→ 감옥으로 이동
→ 돈 생성장소에 돈 생성
6.4 돈 생성장소
역할

죄수 처리 완료 후 돈이 생성되는 장소이다.

동작
죄수 처리 완료
→ 돈 생성장소에 돈 아이템 생성
→ 플레이어가 접근
→ 돈이 플레이어 등으로 흡수
→ 플레이어의 돈 적재량 증가

돈은 플레이어가 등에 직접 짊어지고, 구매칸에서 사용한다.

6.5 구매칸
역할

돈을 투입해 특정 기능을 해금하거나 업그레이드하는 구역이다.

공통 동작
플레이어가 돈을 들고 구매칸에 진입
→ 돈이 하나씩 구매칸으로 흡수
→ 남은 비용 감소
→ 비용을 모두 지불하면 구매 완료
→ 지정된 기능 실행
구매칸 종류
구매칸	결과
무기 강화 구매칸	플레이어 채굴 데미지/스윙 길이/1회 타격 수/박스 범위/등 광석 한도 동시 변동, 무기 외형 교체 (3단계 누적)
광부 일꾼 구매칸	광부 일꾼 3명 동시 생성 (1회성 구매, 구매 후 칸 제거)
죄수 처리 일꾼 구매칸	죄수 처리 일꾼 생성 (영상에 미등장, MVP 제외 가능)
감옥 확장 구매칸	감옥 수용 인원 증가 + 확장 시네마 (1회성 구매, 게임 엔드 트리거)

역할

주요 속성

흐름

→ 현재 수용 인원 증가

8. 오브젝트 풀 설계

본 게임은 반복적으로 생성/제거되는 오브젝트가 많기 때문에, Instantiate와 Destroy를 직접 반복하지 않고 오브젝트 풀을 사용한다.

8.1 풀링 대상
풀 대상	사용처
광석 아이템	채굴, 흡수, 컨테이너 투입 연출
수갑 아이템	수갑 출력, 수집, 죄수 지급 연출
돈 아이템	돈 생성, 수집, 구매 투입 연출
죄수	죄수 대기열, 감옥 이동
광부 일꾼	구매 후 생성
죄수 처리 일꾼	구매 후 생성
채굴 이펙트	광석 타격 시
흡수 이펙트	자원이 대상에게 빨려 들어갈 때
플로팅 텍스트	비용, MAX, 보상 표시
8.2 풀 매니저 역할
기능	설명
사전 생성	시작 시 필요한 수량만큼 오브젝트 생성
Spawn	비활성 오브젝트를 가져와 활성화
Despawn	사용 완료된 오브젝트를 비활성화
자동 확장	풀 부족 시 추가 생성 가능
타입별 관리	자원/캐릭터/이펙트 별로 풀 분리
8.3 풀링 설계 방향

각 아이템은 공통적으로 다음 흐름을 가진다.

Pool에서 꺼냄
→ 위치/목표 설정
→ 이동 또는 연출 실행
→ 목적지 도달
→ 실제 수량 반영
→ Pool로 반환

예시:

상황	연출

9.3 구현 방침 (영상 분석 후 합의)
패턴 — MVC + MVP 혼합. View 는 passive (인스펙터 슬롯 + dumb 메서드 노출), Controller(여기선 *System POCO)가 Model 변형. 단일 Update 흐름은 Player Mono facade 가 Tick 위임.
POCO 우선 — 로직 클래스는 가급적 plain C#. Unity 콜백(OnTriggerStay, OnEnable 등) 또는 [SerializeField] 인스펙터 노출이 필수일 때만 MonoBehaviour. Mono 컴포넌트가 한 GameObject 에 덕지덕지 붙는 것 회피.
엔티티 facade — 캐릭터별로 Mono 한 개(예: Player)가 POCO 서브시스템(MovementSystem, MiningSystem, StackVisualizer 등)을 보유하고 Update 를 위임. 인스펙터 슬롯도 facade 한 곳에 모음.
DI — SystemManager 가 모델/매니저 생성 후 엔티티에 Init() 으로 주입. Awake 에서 모델만 만들고, 엔티티 주입은 Start 에서 (Awake 끝난 뒤를 보장). Zenject 같은 컨테이너 미사용.
반응성 — 자체 ReactiveProperty<T> + EventChannel<T>. UniRx/R3 등 외부 의존 없음. 부족 해지면 R3 로 교체.
비동기 — UniTask. 코루틴 대신 시퀀서/딜레이/취소.
이동 — NavMesh (com.unity.ai.navigation 1.1.5). Player 는 agent.Move() 로 조이스틱 입력 직접 반영, AI 는 agent.SetDestination().
명명 컨벤션 — 클래스/공개 멤버 PascalCase, private 필드/로컬 camelCase, 매개변수는 _camelCase (밑줄 prefix).
폴더 구조 — Assets/Project/Scripts/ 아래 Reactive / Core / Configs / Models / Managers / Movement / Input / Controllers (Player, Prisoner) / Facilities / View (World 포함, 카메라/빌보드도 여기) / Entities / Game 로 분리.
시설 prefab 구성 원칙 — 시설이 단일 zone (입력 또는 출력 1개) 만 가지면 ResourceInputZone/OutputZone 과 같은 GameObject 에 합쳐 (RequireComponent), 입력+출력처럼 공간적으로 분리되는 zone 이 2개 이상이면 자식 GameObject 로 분리. 자체 stack anchor 필드는 두지 않고 zone.transform 을 stack 시각 위치로 사용.

18. 최종 구현 목표 요약

이 과제에서 최종적으로 보여줘야 할 것은 다음이다.

플레이어가 이동한다.
광석을 캔다.
광석을 등에 쌓는다.
광석을 컨테이너에 넣는다.
컨테이너가 수갑을 만든다.
수갑을 손에 든다.
죄수에게 수갑을 준다.
죄수가 감옥으로 이동한다.
돈이 생긴다.
돈을 등에 쌓는다.
돈으로 장비를 강화하거나 일꾼을 고용하거나 감옥을 확장한다.
